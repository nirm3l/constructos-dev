# Projects

A **project** is the top-level container for everything your team works on. Tasks, notes, specifications, and rules all live inside a project. Think of it as a dedicated workspace for a product, service, or initiative.

---

## Creating a project

Click the **+** button in the projects sidebar. Two paths:

- **Start from a template** — pre-fills example tasks, specs, and rules so you have something to work with immediately.
- **Start blank** — a clean project you configure from scratch.

### Starter templates

| Template | Best for |
|---|---|
| Web App | SaaS products, dashboards, admin tools, customer portals |
| API Service | Backend services, workers, integration pipelines |
| DDD System | Domain-driven design with bounded contexts and aggregates |
| Web Game | Browser games, interactive experiences |
| Blank | Anything that doesn't fit the above |

> **Tip:** Even if you're starting blank, browse the starter templates first. They give you a good sense of how to structure tasks and specs for different types of projects.

## Setting up your board

The first thing to configure is your **statuses** — they become the columns on your task board.

Go to **Project Settings → Statuses** and define the stages work moves through in your team. A few common patterns:

| Simple | Engineering | Design & build |
|---|---|---|
| To Do | Backlog | Brief |
| In Progress | In Dev | Design |
| Done | In Review | Dev |
| | QA | QA |
| | Done | Shipped |

> **Tip:** Fewer statuses is usually better. Add a new column only when you genuinely need to distinguish that stage from the ones around it.

## Project Settings at a glance

| Setting | What it does |
|---|---|
| **Name & description** | Label and one-liner for the project |
| **Statuses** | The columns on your task board |
| **External links** | Attach URLs (repo, design file, docs) for quick access |
| **Rules** | Short guidelines for the team and AI agent |
| **AI chat indexing** | Whether the in-project chat can search your notes and specs |

---

## Members and roles

Open **Project Settings → Members** to invite your team.

| Role | What they can do |
|---|---|
| **Owner** | Full control, including deleting the project |
| **Admin** | Manage settings, members, and all content |
| **Member** | Create and edit tasks, notes, and specs |
| **Viewer** | Read-only — can see everything but not change it |

---

## Archiving a project

When a project is complete (or on hold), archive it from Project Settings. Archived projects:
- Disappear from the active sidebar.
- Retain all data — tasks, notes, specs, history.
- Can be restored any time from the archived projects view.

Deleting a project is permanent. Archive first if you're unsure.
